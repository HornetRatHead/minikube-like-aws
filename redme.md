================================================================================
           STOCKWISE: MLOPS INFRASTRUCTURE ON KUBERNETES (MINIKUBE)
================================================================================

1. ОПИС ПРОЄКТУ
--------------------------------------------------------------------------------
MLOps платформа, розгорнута в Minikube.
Iмітує хмарну інфраструктуру AWS (EKS, RDS, S3) для повного циклу 
життєвого циклу ML-моделей.

2. СТРУКТУРА ТА ТЕХНОЛОГІЇ
--------------------------------------------------------------------------------
- Оркестрація: Kubernetes (Minikube)
- IaC:         Terraform (Helm, Secrets, Namespaces)
- ML Tracking: MLflow (Tracking Server + Auth)
- Database:    PostgreSQL (Емуляція RDS)
- Artifacts:   SeaweedFS (Емуляція S3)
- CD/GitOps:   ArgoCD

3. ШВИДКИЙ ЗАПУСК
--------------------------------------------------------------------------------
Запустіть автопілот:
   chmod +x ./scripts/setup.sh
   ./scripts/setup.sh

4. ЕНДПОЇНТИ (ПІДКЛЮЧЕННЯ ЗОВНІ)
--------------------------------------------------------------------------------
Ці адреси доступні з вашої основної ОС (Ubuntu) після роботи setup.sh:

- MLflow UI:  http://localhost:8080 (User: user)
- ArgoCD UI:  https://localhost:8443 (User: admin)
- S3 API:     http://localhost:8333 (Endpoint для boto3/mlflow)
- Postgres:   localhost:5432       (Host для DB-клієнтів)

5. ВНУТРІШНІ ЕНДПОЇНТИ (ДЛЯ K8S PODS)
--------------------------------------------------------------------------------
Використовуйте ці DNS-імена для зв'язку між сервісами всередині кластера:

- MLflow:   mlflow-tracking.application.svc.cluster.local:80
